# Iteration 11: Discoverable Drill-Down Commands

## Goal

Add contextual follow-up command hints to JSON and text output, so LLM agents (and humans) can discover how to drill down from summary data without needing to know the full command tree. The HATEOAS principle applied to CLI output.

## Design

### Hint generation

Hints are **concrete, runnable commands** — never templates with placeholders. The `generate_hints()` function inspects the actual output data (e.g. the top tags, the property names present) and builds real commands an agent can copy-paste or execute directly.

Key design decisions:

- **Concrete-only hints (no templates):** Every hint is a valid, executable command string. No `<placeholder>` syntax that an LLM would need to fill in. This is safer for automated execution and avoids hallucination risks.
- **Opt-in `--hints` flag:** Hints are not shown by default. The `--hints` flag enables them. This keeps default output clean for scripting and backward-compatible.
- **JSON envelope:** When hints are enabled, JSON output wraps in `{"data": ..., "hints": [...]}`. The original output goes into `data` unchanged, and hints are a flat string array.
- **Text arrow prefix:** In text mode, hints appear as `-> hyalo ...` lines after the main output.
- **Suppress with `--jq`:** When `--jq` is active, hints are suppressed — the user is already doing custom extraction and hints would interfere with the jq filter.
- **Flag propagation:** If `--dir` or `--glob` was passed, hint commands include those flags too, so the suggested commands work in the same context.
- **State-aware generation:** Hints are generated by inspecting the actual output data (e.g. which tags are most used, which properties exist), not from static lookup tables. This means hints are relevant to what the user actually sees.

### Tags summary sort bug

During dogfooding, discovered that hints for the tags summary were showing the alphabetically first tags instead of the most-used tags. The tags summary output was sorted alphabetically, but the hint generator assumed frequency order. Fixed by ensuring the hint generator sorts by count descending before picking top entries.
